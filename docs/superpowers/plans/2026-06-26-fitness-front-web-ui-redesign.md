# Fitness Front Web UI 风格迭代设计方案

> 版本：v1.0  
> 日期：2026-06-26  
> 适用范围：fitness-front-web 用户端

---

## 一、设计概述

基于参考设计图的视觉风格分析，对 fitness-front-web 项目进行整体 UI 风格迭代。目标是从当前偏工具型的柔和风格，转变为充满力量感、专业健身品牌感的视觉风格。

### 1.1 设计关键词
- 力量感 / Power
- 运动张力 / Dynamic
- 专业健身 / Fitness Professional
- 高对比 / High Contrast
- 简洁硬朗 / Clean & Bold

### 1.2 与现有设计的核心差异

| 维度 | 当前设计 | 新设计 |
|------|---------|--------|
| 主色调 | 紫色渐变 (`#667eea → #764ba2`) | 健身红 (`#D93831`) |
| 导航栏 | 白色背景 + 灰色文字 | 黑色背景 + 白色文字 + 红色高亮 |
| 卡片圆角 | 12px 大圆角 | 4px 小圆角（硬朗感） |
| 标题风格 | 普通文字 + emoji | 粗体大写 + 红色斜杠装饰 |
| 页面背景 | 浅灰 `#f5f7fa` | 纯白 `#FFFFFF` |
| 整体感觉 | 柔和、偏工具型 | 力量感、运动感、专业健身品牌感 |

## 三、字体系统

### 3.1 字体家族

使用系统无衬线字体栈，保证跨平台一致性和加载性能：

```css
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto,
  'Helvetica Neue', Arial, 'Noto Sans', sans-serif;
```

### 3.2 字号层级（Type Scale）

基于 14px 基准字号，采用 1.25 比例因子：

| 层级 | 字号 | 行高 | 字重 | 用途 |
|------|------|------|------|------|
| **Hero 大标题** | 48px (3rem) | 1.2 | 700 (Bold) | 首屏大标语，全部大写 |
| **页面主标题 H1** | 32px (2rem) | 1.3 | 700 (Bold) | 各区块标题，带红色斜杠装饰 |
| **次级标题 H2** | 24px (1.5rem) | 1.3 | 600 (Semibold) | 页面内大模块标题 |
| **卡片标题 H3** | 18px (1.125rem) | 1.4 | 600 (Semibold) | 卡片、模块标题 |
| **正文 Body** | 14px (0.875rem) | 1.6 | 400 (Regular) | 主要内容文字 |
| **小字/辅助 Caption** | 12px (0.75rem) | 1.5 | 400 (Regular) | 标签、元信息、说明文字 |

### 3.3 字重使用规范

- **700 Bold**：仅用于大标题和需要强调的关键词
- **600 Semibold**：用于卡片标题、小节标题
- **500 Medium**：用于按钮文字、菜单文字
- **400 Regular**：正文、辅助文字（默认）

## 四、间距与留白系统

### 4.1 间距基准

采用 **8px 基准网格**，所有间距均为 8 的倍数，确保视觉节奏一致。

| 级别 | 数值 | 用途示例 |
|------|------|---------|
| xs | 4px | 元素内微调、图标间距 |
| sm | 8px | 紧凑元素间距、标签间距 |
| md | 16px | 卡片内边距、表单元素间距 |
| lg | 24px | 区块内间距、模块间距 |
| xl | 48px | 大区块之间的间距 |
| 2xl | 80px | 页面一级区块之间的垂直间距 |

### 4.2 布局容器

- **最大内容宽度**：1200px
- **页面左右内边距**：24px（移动端 16px）
- **区块顶部内边距**：80px
- **区块底部内边距**：80px

## 五、组件样式规范

### 5.1 按钮 Button

#### 主按钮（Primary）

- 内边距：`12px 32px`
- Hover：背景 `#D93831`，文字白色

#### 文字按钮（Text）
- 背景：透明
- 文字：`#D93831`
- 无边框无背景
- Hover：文字加深，加下划线

### 5.2 卡片 Card

- 背景：`#FFFFFF`
- 边框：1px solid `#E5E7EB`
- 圆角：4px
- 内边距：24px
- 阴影：默认无阴影
- Hover 阴影：`0 8px 24px rgba(0, 0, 0, 0.08)`
- Hover 位移：`translateY(-4px)`
- 过渡：`all 0.3s ease`

### 5.3 导航栏 Navigation

#### 顶部信息条（Top Bar）
- 高度：36px
- 背景：`#D93831`
- 文字：白色 12px
- 内容：左侧电话/邮箱，右侧社交媒体图标

#### 主导航栏（Main Nav）
- 高度：64px
- 背景：`#0A0A0A`（纯黑）
- Logo：白色文字 + 红色哑铃图标，字号 20px，字重 700
- 菜单文字：白色 14px，字重 500
- 当前菜单：红色 `#D93831`
- Hover 菜单：红色 `#D93831`
- 用户区域：白色头像 + 白色文字

### 5.4 标题装饰（Section Title）

每个区块标题采用统一的标志性设计：

1. **红色双斜杠前缀**：标题左侧有两条红色斜杠 `//`，呈倾斜状
2. **双色标题**：标题前半部分深色/白色，后半部分红色
   - 例：`ABOUT US` → ABOUT (深色) + US (红色)
   - 例：`OUR COURSES` → OUR (深色) + COURSES (红色)
3. **居中对齐**：区块标题居中，下方有描述文字

### 5.5 输入框 Input

- 高度：36px
- 边框：1px solid `#DCDFE6`
- 圆角：2px
- 背景：白色
- 文字：`#333333`，14px
- Focus：边框 `#D93831`，无阴影或极淡红色阴影
- Placeholder：`#C0C4CC`

## 七、标志性设计元素（Signature Elements）

### 7.1 红色双斜杠标题装饰
每个区块标题前的 `//` 红色斜杠，是整个设计系统的核心识别符号，贯穿所有页面。

### 7.2 双色标题
标题中关键词使用红色强调，形成视觉焦点，强化品牌记忆。

### 7.3 圆形人物蒙版
人物图片使用正圆形红色背景衬托（参考 ABOUT US 区域），增加设计趣味。

### 7.4 图片覆盖式卡片
课程/项目卡片以图片为背景，底部叠加白色文字标题，营造杂志感。

### 7.5 回到顶部按钮
右下角红色方形按钮，白色向上箭头，固定悬浮。

## 十、微交互与动效规范

### 10.1 动效设计原则

- **快速**：所有动效 ≤ 300ms，不拖沓
- **有意义**：动效传达状态变化，非纯粹装饰
- **一致**：同类交互使用相同动效曲线

### 10.2 全局动效定义

```css
:root {
  --ease-out: cubic-bezier(0.25, 0.46, 0.45, 0.94);
  --ease-in-out: cubic-bezier(0.42, 0, 0.58, 1);
  --transition-fast: 150ms var(--ease-out);
  --transition-normal: 250ms var(--ease-out);
  --transition-slow: 400ms var(--ease-in-out);
}
```

### 10.3 场景动效清单

| 场景 | 动效 | 时长 | 曲线 | 说明 |
|------|------|------|------|------|
| 统计数字 | count-up 从 0 滚到目标值 | 600ms | ease-out | 页面加载时触发 |
| 卡片 hover | `translateY(-4px)` + `box-shadow` 扩散 | 250ms | ease-out | 已有上浮，需加阴影扩散 |
| 页面切换 | `<router-view>` fade | 200ms | ease-out | opacity 0→1 |
| 进度条 | 从左到右填充 | 500ms | ease-out | `el-progress` 默认即此 |
| 按钮点击 | `scale(0.97)` 按压反馈 | 100ms | ease-out | 触感回馈 |
| Hero 区背景 | Ken Burns 微缩放 | 8s | ease-in-out | 无限循环，scale(1) → (1.05) |
| 列表项进入 | staggered fade-in（逐个淡入） | 300ms × index | ease-out | 50ms stagger delay |
| 对话框打开 | `scale(0.95) → scale(1)` + fade | 200ms | ease-out | 从中心微缩放大 |
| 删除操作 | 条目向右滑出 + fade | 200ms | ease-in | 给删除操作空间感知 |

### 10.4 Vue Router 过渡

```vue
<!-- App.vue -->
<router-view v-slot="{ Component }">
  <transition name="page-fade" mode="out-in">
    <component :is="Component" />
  </transition>
</router-view>
```

```css
.page-fade-enter-active, .page-fade-leave-active { transition: opacity 0.2s ease; }
.page-fade-enter-from, .page-fade-leave-to { opacity: 0; }
```

## 十一、中文字体排版适配

### 11.1 字体栈

中文优先使用系统自带中文字体，避免额外加载字体文件：

```css
font-family: -apple-system, BlinkMacSystemFont, 'PingFang SC', 'Microsoft YaHei',
  'Hiragino Sans GB', 'Noto Sans CJK SC', 'WenQuanYi Micro Hei',
  sans-serif;
```

### 11.2 中文排版调整

| 项目 | 英文规范 | 中文调整 | 原因 |
|------|---------|---------|------|
| 正文行高 | 1.6 | **1.5** | 中文无 ascender/descender，行高过大显得松散 |
| 英文/数字行高 | — | **1.6** | 混合内容保持英文原有行高 |
| Bold 字重 | 700 | **700**（明确指定） | 系统字体栈下中文 700 不一定等于 Bold，需显式声明 |
| 标题字间距 | 0 | **0.02em** | 中文标题稍加字间距可增加力量感 |
| 中英文/数字混排间距 | — | 自动（PingFang SC 支持） | 中文与英文/数字之间应有 1/4 em 间隙 |

### 11.3 双色标题：中文版

方案文档英文示例 `ABOUT US` ↔ `ABOUT`(黑) + `US`(红) 对中文需调整：

**中文适配方案：后 N 个字变红**

```
"热门课程" → "热门"（黑） + "课程"（红）
"训练记录" → "训练"（黑） + "记录"（红）
"饮食计划" → "饮食"（黑） + "计划"（红）
"身体数据" → "身体"（黑） + "数据"（红）
```

**实现方式：** 提供 `<SectionTitle>` 通用组件，props 传入完整标题 + 红色字数：

```vue
<SectionTitle full="热门课程" :redCount="2" />
<!-- 渲染: <h2>热门<span class="red">课程</span></h2> -->
```

### 11.4 中文字号微调

| 层级 | 英文规范 | 中文实际 | 说明 |
|------|---------|---------|------|
| Hero 大标题 | 48px(3rem) | **44px** | 中文字形视觉占比较大，略缩小 |
| 页面主标题 | 32px(2rem) | **30px** | 同上 |
| 次级标题 | 24px(1.5rem) | **22px** | 同上 |
| 卡片标题 | 18px | **18px** | 保持 |
| 正文 | 14px | **14px** | 保持 |
| 小字 | 12px | **12px** | 保持 |

## 十二、健身平台专属视觉元素

### 12.1 数据可视化升级

当前 `el-table` + 纯文字展示营养/训练数据过于朴素。需增加图表组件：

| 页面 | 建议图表 | 推荐库 | 说明 |
|------|---------|--------|------|
| BodyMetrics 体重趋势 | 折线图（7/30 天） | `@element-plus/icons-vue` 或 Chart.js | 目前只显示表格，加一个趋势卡 |
| Diet 宏量素占比 | 环形图（蛋白:碳水:脂肪） | 自定义 SVG ring 或 ECharts | 每日汇总下方，直观展示三大营养素比例 |
| Training 热量统计 | 柱状图（7 天汇总） | ECharts | 日期为横轴，热量为纵轴 |
| 统计卡片 | 环形进度条 | `el-progress` type="circle" | 替代目前直线进度条，更紧凑有力 |

### 12.2 图标系统

当前大量使用 emoji（🍎🍗📚💪📊🎯🌙☀️🍪🌅）。在新设计风格下应替换为专业图标：

| 场景 | 当前 | 建议替换 |
|------|------|---------|
| 页面标题 | Emoji 前缀 | 移除 emoji，纯文字 + 红色斜杠装饰 |
| 餐型标签 | Emoji 🌅☀️🌙🍪 | Element Plus 图标（`Sunrise`/`Sunny`/`Moon`/`Dessert`） |
| 功能入口卡 | 大号 emoji | SVG 自定义图标或 Element Plus 图标 |
| 统计卡片 | 无图标 | 每张卡加小图标在数值旁 |

### 12.3 营养目标可视化

Diet 页面目前用直线进度条展示目标进度。健身 App 更常见的做法：

```
┌──────────────────────────┐
│    ┌──────────┐          │
│    │  环形图    │  1850   │
│    │  74%      │  kcal   │
│    └──────────┘          │
│   目标 2500  ·  剩余 650  │
└──────────────────────────┘
```

四个宏量素用 4 个小型环形图排列，比直线进度条视觉冲击力强得多。

## 十三、关键复杂组件设计补充

### 13.1 食物搜索对话框（Diet 页面）

- **快捷标签**：红底白字小胶囊 → 改为浅灰底深灰字小胶囊，hover 红底白字

### 13.2 餐型 Tab 栏（Diet 页面）

- 当前使用 `el-tabs type="card"`（卡片式）→ 改为下划线式
- 激活态下划线：2px solid `#D93831`

## 十四、移动端导航方案

### 14.1 底部 TabBar（< 768px）

健身场景下，用户常单手操作，底部 TabBar 比汉堡菜单更实用：

```
┌──────────────────────────────────┐
│         页面内容区域              │
├──────┬──────┬──────┬──────┬──────┤
│  🏠  │  📚  │  🍎  │  🏋  │  👤  │
│ 首页 │ 课程 │ 饮食 │ 训练 │ 我的 │
├──────┴──────┴──────┴──────┴──────┤
│         系统导航栏                │
└──────────────────────────────────┘
```

- 高度 56px
- 背景 `#FFFFFF`，顶部 1px `#E5E7EB` 分割线
- 激活态：图标 + 文字变红 `#D93831`
- 非激活态：图标 + 文字 `#999999`
- 字号 11px
- 5 个核心入口：首页、课程、饮食、训练、我的（去掉 AI 聊天和教练面板，这两项收入"我的"页面）

### 14.2 桌面端保留顶导航

≥ 768px 时，底部 TabBar 隐藏，使用文档定义的黑底顶导航。

## 十五、空状态与加载态

### 15.1 Skeleton Screen（骨架屏）

替代 `v-loading` spinner 的闪烁加载效果。每个数据密集型页面使用对应形状的灰块占位：

| 页面 | 骨架形状 |
|------|---------|
| CourseList | 3×2 卡片灰色矩形（160px 高 × 满宽） |
| Diet | 4 个统计卡片 + 食物列表灰色条 |
| Training | 3 个统计卡片 + 表格灰色条 |
| BodyMetrics | 4 个统计卡片 + 图表灰色矩形 |

```css
.skeleton {
  background: linear-gradient(90deg, #F0F0F0 25%, #E4E4E4 50%, #F0F0F0 75%);
  background-size: 200% 100%;
  animation: skeleton-loading 1.5s ease-in-out infinite;
}
@keyframes skeleton-loading {
  0%   { background-position: 200% 0; }
  100% { background-position: -200% 0; }
}
```

### 15.2 品牌化空状态

当前 `el-empty` 使用默认灰色插图，缺乏品牌感。建议：

- 4 个场景化空状态 SVG 插图：
  - 饮食：空餐盘图案 → "今日暂无记录，添加你的第一餐吧"
  - 训练：空哑铃图案 → "还没有训练记录，开始第一次训练"
  - 课程：空播放器图案 → "暂未订阅课程，去课程中心看看吧"
  - 身体数据：空体脂夹图案 → "还未录入身体数据"
- 插图尺寸 120px，下方红色文字按钮引导操作

## 十六、深色区块使用边界

深色背景（`#0A0A0A`）在全站中的使用限定：

| 区块 | 背景色 | 说明 |
|------|--------|------|
| 顶部信息条 | `#D93831` | 红色，最高层级 |
| 主导航栏 | `#0A0A0A` | 黑色，固定顶部 |
| **Hero 区（仅首页）** | `#0A0A0A` | 黑色全宽，唯一使用深色的内容区 |
| 所有功能页面内容 | `#FFFFFF` | 纯白，数据密集型页面不适用深色背景 |
| 次级区块 | `#F8F9FA` | 浅灰，用于卡片底色或区分区块 |
| CTA 横幅 | `#0A0A0A` | 黑色，首页底部一处 |
| 页脚 | `#0A0A0A` | 黑色全宽 |

**核心原则：一个页面最多出现 2 个深色区块（导航栏不计入）。首页以外不应有深色内容区。**
